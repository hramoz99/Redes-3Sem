GRUPOS 
 Administradores de servico - responsavel por manter e oferecer servicos de dominio ADDS , incluindo o gerenciamento de controladores de dominio e a configuração do ADDS

 Adminsitradores de dados: Responsavel pela manutenção dos dados armazenados em adições eem estações de trabalho  e servidores menbro do dominio.


Os grupos são usados para coletar contas de usuario, contas de computador e outros grupos em unidades gerenciaveis. Trabalhar com grupos em vez de com usuarios individuais ajuda a simplificar a manutenção

 Há dois tipos de grupos AD:

 Grupos de distribuição usado para criar listas de distribuição de e-mail
 Grupos de segurança usado para atribuir permissoes a recursos compartilhados


Atribua direitos de usuario a grupos de seguranca no AD
 Os direitos do usuario são atribuidos a um grupo de seguranca para determinar o que os membros desse grupo podem fazer dentro do escopo de um dominio ou floresta. Um usuario adicionado ao grupo operadores de backup no AD tem a capacidade de fazer backup e restaurar arquivos e diretorios localizados em cada comtrolador de dominio no dominio


Atribua permissoes a grupos de seguranca para recursos
 As permissoes sao diferentes dos direitos do usuario. As permissoes sao atribuidas ao grupo de seguranca para o recurso compratilhado. As permissoes determinam quem pode acessar o recurso e o nivel de acesso, como controle total.



Escopos

 Os grupos sao caracterizados por um escopo que identifica a extensao a qual o grupo e aplicado na arvore de dominio ou floresta. o escobo do grupo define onde o grupo pode receber permissoes.


 .Universal
 .Global
 .Dominio local

 
Grupos de identidades especiais
 grupos de identidade especiais noa tem associações especificas que podem ser modificadas, mas elas podem representar usuarios diferentes em momento diferentes, dependendo das circunstancias
 Alguns desses grupos incluem proprietario criador, lote e usuario autenticado


 Grupos de seguranca padrao

 Os grupos padrao, como o grupo Domain Admins sao grupos de seguranca criados automaticamente quando cria dominio do active Directory
 Voce pode usar esse grupos predefinidos para ajudar a controlar o acesso a recursos compartilhados e para delegarfu funções administrativas especificas em todo o dominio
 os grupos padrao estao localizados no conteiner builtin e no conteiner usuarios em usuarios e computadores do active directory
 O conteiner interno inclui grupos que sao definidos com o escopo de dominio local. Os usuarios incluem grupos que sao definidos com escopo global e grupos definidos com escopo de dominio local
 Voce pode mover grupos localizadfos nesses recipientes para outros grupos ou unidades organizacionais  do dominio mas nao podemmove-los para outros dominios

O descritor de seguranca esta presente no obejto ADminSDHOlder

Isso significa que se voce quiser modificar as permissoes em um dos grupos de administradores de serviços ou em qualquer uma de suas contas de mebros sera necessario modificar o descritos de seguranca no objeto AdminSDHolder para que ele seja aplicado de forma conscistente. tenha cuidados ao fazer essas modificações porque voce tambem esta alterando as configurações padrao que serao aplicadas a todas as suas contas administrativas protegidas

O grupo operadores de conta concede privilegios de criaçao de conta limitados a um ususario

Os membros desse grupo podem criar e modificar a maioria dos tipos de conta, incluindo os usuarios, grupos locais e grupos globais e os membros podem fazer logon localmente em controladores de dominio

Os membros do grupo operadores de conta nao podem gerenciar a conta de ususario administrador, as contas usuarios de administradores ou grupos administradores, operadores de servidor, operadores de conta, operadores de backup ou operadires de impressao. os membros desse grupo nao mpodem modificar direitos de usuario

O grupo operadores de conta aplica-se as versoes do sistema operacional windows server listadas na tabela de grupos de seguranca padrao do active directory



Administradores 
Os membros do grupo administradores tem acesso completo e irrestrito ao computador ou se o computador for promovido a um controlador de dominio os membros terao acesso irrestrito do dominio

O grupo administradores aplica-se as verspoes do sistema operacional windows server listadas na tabela de grupos de seguranca padrao do active directory

A associação pode ser modificada poir membros dos grupos de administradores de serviço padrao administradores de dominio no dominio ou administradores 

Esse grupo tem privilegio especial para apropriar se de qualquer objeto no diretorio ou em qualquer recursos em um controlador de dominio

Essa conta é considerada um grupo de administradores de serviços pois seus membros tem acesso completo ao controladores de dominio no dominio

22. Delegação de documentos para Active Directory

A melhor maneira de controlar o acesso ao Active Directory e recursos relacionados é usar grupos de segurança.

Se estiver delegando direitos a indivíduos, você está perdendo o controle de quem tem acesso.

Crie grupos personalizados com nomes muito específicos, documente quem tem direitos e um processo para adicionar novos usuários.

Não se limite a permitir que usuários sejam adicionados a esses grupos personalizados sem um processo de aprovação. Esta é apenas outra maneira pela qual as permissões podem ficar fora de controle.

Saiba quais grupos são delegados a quais recursos, documente e certifique-se de que sua equipe esteja na mesma página.
